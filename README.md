Task 3: Database Migration 

Database migration is a high-stakes process where data is moved between different environments, such as from a local MySQL development server to a PostgreSQL production server.

Schema Mapping: I focused on translating data types and constraints to ensure that the "Logical Blueprint" of the bookstore remained intact during the transfer.

Integrity Constraints: During the migration, I prioritized Referential Integrity. This ensures that every order migrated still points to a valid customer and a valid book, preventing "orphaned records" that could crash the application. This task proved that a well-designed schema is scalable and portable across different SQL platforms
